
# תוכנית: מודל קבלת פנים משופר לאחר התחברות

## סקירה כללית
במקום המודל הנוכחי שמכריח את המשתמש להזין שם (מה שמציק), ניצור מודל קבלת פנים ידידותי שיציג:
1. הודעת הצלחה על התחברות
2. ברכה אישית עם השם מ-Google (שכבר בעברית אם המשתמש הגדיר כך ב-Google)
3. הצגה קצרה של האפליקציה ויכולותיה
4. כפתור "בואו נתחיל!" לסגירת המודל

## מה ישתנה

### השם - אוטומטי, בלי להזין
- נשתמש ב-`user.user_metadata.full_name` מ-Google
- אם המשתמש הגדיר את השם בעברית בחשבון Google, זה מה שיופיע
- נשמור את השם אוטומטית ב-localStorage ללא צורך בהזנה ידנית

### המודל החדש יכלול
1. **אנימציית הצלחה** - וי ירוק עם קונפטי/ניצוצות
2. **ברכה אישית** - "שלום [שם מ-Google]!" 
3. **הצגה קצרה של האפליקציה**:
   - כרטיסים יפים עם אייקונים
   - הסבר על יצירת רשימות קניות
   - סנכרון בין מכשירים
   - תובנות כלכליות
4. **כפתור אחד בלבד** - "בואו נתחיל!" (בלי להזין כלום)

## שינויים טכניים

### קובץ: `src/components/WelcomeNameModal.tsx`
שינוי מלא של הקומפוננטה:

**מה יוסר:**
- שדה הזנת שם (Input)
- כפתור "אולי אחר כך"
- ה-state של name ו-isSubmitting
- הטופס (form)

**מה יתווסף:**
- שלב ראשון: אנימציית הצלחה עם וי ירוק וקונפטי (2 שניות)
- שלב שני: מסך הצגה עם:
  - ברכה אישית: "שלום, [שם]! 🎉"
  - כותרת: "ברוכים הבאים לעגליסט"
  - 3 כרטיסי יתרונות עם אייקונים:
    - עגלה: "צור רשימות קניות בקלות"
    - ענן: "הרשימות שלך שמורות ונגישות מכל מכשיר"  
    - גרף: "קבל תובנות על הקניות שלך"
  - כפתור: "בואו נתחיל!"

**לוגיקה חדשה:**
```tsx
// שמירה אוטומטית של השם מ-Google
useEffect(() => {
  if (user?.user_metadata?.full_name) {
    localStorage.setItem('user_display_name', user.user_metadata.full_name);
  }
}, [user]);
```
